# Grocery Store Management System using SQL

## 📌 Project Overview
The Grocery Store Management project is a SQL-based data analysis project designed to analyze customer behavior, product performance, sales trends, supplier contribution, and employee performance in a grocery store environment.

The project uses a relational database structure with multiple interconnected tables to generate meaningful business insights from transactional data.

---

## 🎯 Objective
The main objective of this project is to transform raw grocery store data into actionable business insights using SQL queries.

This project helps in:
- Understanding customer purchasing behavior
- Identifying top-selling products
- Analyzing sales and revenue trends
- Evaluating supplier contributions
- Measuring employee performance
- Supporting data-driven business decisions

---

## 🏪 Business Problem
The grocery store lacks a structured system to analyze sales, customer activity, product performance, supplier contribution, and operational efficiency.

Due to the absence of proper data analysis, the business faces difficulties in:
- Identifying high-value customers
- Tracking best-selling products
- Understanding monthly sales trends
- Evaluating supplier and employee performance
- Optimizing inventory and sales strategies

This project addresses these challenges using SQL-based analytical queries.

---

## 🗂️ Database Schema
The project contains 7 relational tables:

- supplier
- categories
- employees
- customers
- products
- orders
- order_details

The database follows relational design principles using:
- Primary Keys
- Foreign Keys
- Cascade Constraints
